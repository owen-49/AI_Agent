## Slide 2 | 项目背景与动机

**标题：** 为什么需要自动化命题？

**痛点：**
- 高等代数命题依赖专家经验，人力成本高
- 传统题库存在题目同质化、答案固化的缺陷
- LLM 直接生成数学题存在"幻觉"风险（逻辑不自洽、数值不友好）

**我们的回答：**
- 将 **LLM 的创造力** 与 **符号计算的严谨性** 结合起来
- 用"反向构造"策略保证题目可解且答案优雅
- 用物理沙箱验证杜绝 AI 幻觉

---

> **演讲要点（45秒）**
> 高等代数教学中有两个现实问题：一是出题高度依赖专家，成本高；二是如果直接用 ChatGPT 出题，它会一本正经地生成一道数学上根本不成立的题目——这就是所谓的 AI 幻觉。
>
> 我们的思路很直接：让 LLM 生成题目草稿，然后用 SymPy 这个符号计算引擎在沙箱里实际跑一遍验证脚本。题目通过了物理验证，才输出给用户。这就是"神经符号"的含义——神经网络负责创造，符号系统负责把关。

## Slide 3 | 系统架构总览

**标题：** 系统架构：三层协同工作流

```
┌─────────────────────────────────────────┐
│           Streamlit Web UI              │
│   参数配置 → 生成触发 → 结果展示/导出    │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│   HigherAlgebraProfessorAgent (核心)    │
│  ┌──────────────────────────────────┐   │
│  │ 随机种子注入 → Prompt 构建        │   │
│  │     ↓                            │   │
│  │ LLM 生成 → JSON 解析             │   │
│  │     ↓                            │   │
│  │ SymPy 验证 ← 不通过则反馈重试     │   │
│  │     ↓ 通过                       │   │
│  │ 入库 + 输出                      │   │
│  └──────────────────────────────────┘   │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│          SQLite 题库持久化               │
└─────────────────────────────────────────┘
```

**关键设计决策：**
- Agent 最多 3 轮自纠正循环
- 验证不通过时，将错误信息作为反馈注入下一轮 Prompt
- 通过则要求特征值为整数（数值优美性检查）

---

> **演讲要点（60秒）**
> 系统分为三层。最上面是 Streamlit 构建的 Web 界面，教师选择知识点和难度，点击按钮即可生成。
>
> 中间是核心的 Agent 层，它执行一个闭环工作流：首先注入随机种子参数——包括矩阵大小、特征值范围、题目类型——来保证每次生成的题目都不一样；然后构建 Prompt 发给 LLM；LLM 返回 JSON，其中包含题干、解析和一段 SymPy 验证脚本；Agent 提取这段脚本在沙箱中实际执行，如果验证失败，就把错误信息反馈给 LLM 重新生成，最多重试 3 轮。
>
> 验证通过后，题目存入 SQLite 数据库，同时在界面上展示。值得一提的是，我们还要求特征值必须为整数——这是一种"数值优美性"过滤，确保题目适合手算。

## Slide 4 | 核心创新：反向构造协议

**标题：** 反向构造 — 从答案出发，保证题目可解

**传统方法（正向）：** 随便写个矩阵 → 求特征值 → 可能是无理数 → 不适合教学

**我们的方法（反向）：** 先定特征值（整数） → 构造对角矩阵 → 乘以随机可逆矩阵 → 得到题目矩阵

**四种构造协议（每次随机选一种）：**

| 协议 | 策略 | 适用题型 |
|------|------|----------|
| A | 对角化逆构造：$A = P \Lambda P^{-1}$ | 特征值/对角化 |
| B | 二次型配方法 | 二次型标准形 |
| C | 特征向量反推：谱分解 | 特征向量分析 |
| D | 初等变换与秩 | 秩与极大无关组 |

---

> **演讲要点（45秒）**
> 这个项目最核心的创新是"反向构造"。常规出题是正向的——写一个矩阵，让学生算特征值——但这样生成的矩阵特征值往往是无理数，不适合考试。
>
> 我们的做法反过来：先选定一组漂亮的特征值，比如 2、5、-3，构造对角矩阵，再乘以一个随机的可逆矩阵 P，得到最终的题目矩阵 A。这样 A 的特征值天然就是我们选的那些整数。题目保证有解，且答案优美。
>
> 我们设计了四种不同的构造协议，每次随机选一种，进一步增加题目的多样性。

## Slide 5 | LLM 幻觉过滤机制

**标题：** 三重质量保障：杜绝 AI 胡说八道

**第一层 — 格式校验：**
- LLM 输出必须是严格 JSON 结构
- 包含 title、latex_statement、sympy_script、analytical_solution 四个字段

**第二层 — 物理沙箱验证：**
- 在受限的 Python 沙箱中执行 sympy_script
- 只暴露 sympy、numpy 等数学库，隔离系统调用
- 检查脚本是否正常执行且产生 result 变量

**第三层 — 数值优美性过滤：**
- 验证通过后，检查特征值是否均为整数
- 无理数特征值 → 反馈 LLM 重新生成

**不通过的处理：**
```
生成的矩阵特征值包含无理数，不符合"数值友好"要求。
请重新构造，确保特征值均为整数。
```
→ 注入下一轮 Prompt，形成自纠正回路

---

> **演讲要点（30秒）**
> 我们设置了三道关卡来防止 AI 幻觉。
> 第一关是格式校验——输出必须是规定的 JSON 结构。
> 第二关是物理沙箱——LLM 生成的验证脚本被放在一个受限的 Python 环境中实际执行，只允许调用 SymPy 和 NumPy，执行失败就直接打回。
> 第三关是美学过滤——即使脚本跑通了，如果特征值不是整数，也会被拒绝并反馈给 LLM 重来。
> 这个反馈闭环是整个系统可靠性的核心。
